## 二、请求层实现（对接后端 Result）

### 2.1 开发代理（vite.config.js）

```js
export default defineConfig({
  plugins: [vue()],
  server: {
    port: 5173,
    proxy: {
      '/api': { target: 'http://localhost:8080', changeOrigin: true }
    }
  }
})
```

> 后端已对 `/**` 全量放开 CORS，也可不用代理直接 `baseURL: 'http://localhost:8080'`。

### 2.2 axios 封装（api/request.js）

统一处理后端 `Result{code,message,data,timestamp}`：

```js
import axios from 'axios'
import { ElMessage } from 'element-plus'
import { useUserStore } from '@/stores/user'
import router from '@/router'

const request = axios.create({ baseURL: '/api', timeout: 10000 })

// 请求拦截：带 token
request.interceptors.request.use(config => {
  const userStore = useUserStore()
  if (userStore.token) config.headers['Authorization'] = 'Bearer ' + userStore.token
  return config
})

// 响应拦截：拆 Result
request.interceptors.response.use(
  resp => {
    const res = resp.data
    if (res.code === 200) return res.data          // 成功直接返回 data
    if (res.code === 401) {                          // 未登录/过期
      useUserStore().logout()
      router.push('/login')
    }
    ElMessage.error(res.message || '操作失败')
    return Promise.reject(res)
  },
  err => { ElMessage.error('网络异常'); return Promise.reject(err) }
)

export default request
```

### 2.3 接口模块示例（api/part.js）

与后端 [PartController](../src/main/java/com/motorparts/controller/PartController.java) 一一对应：

```js
import request from './request'
```

## 三、状态管理与登录态（Pinia）

### 3.1 用户 Store（stores/user.js）

```js
import { defineStore } from 'pinia'
import { authApi } from '@/api/auth'

export const useUserStore = defineStore('user', {
  state: () => ({
    token: localStorage.getItem('token') || '',
    roles: JSON.parse(localStorage.getItem('roles') || '[]'),
    userInfo: {}
  }),
  getters: {
    isAdmin:       s => s.roles.includes('admin'),
    isPurchase:    s => s.roles.includes('purchase'),
    isRequisition: s => s.roles.includes('requisition'),
  },
  actions: {
    async login(form) {
      const data = await authApi.login(form)   // {token, roles, userInfo}
      this.token = data.token
      this.roles = data.roles
      this.userInfo = data.userInfo
      localStorage.setItem('token', data.token)
      localStorage.setItem('roles', JSON.stringify(data.roles))
    },
    logout() {
      this.token = ''; this.roles = []; this.userInfo = {}
      localStorage.clear()
    }
  }
})
```

### 4.2 全局路由守卫

```js
router.beforeEach((to, from, next) => {
  const userStore = useUserStore()
  if (to.path === '/login') return next()
  if (!userStore.token) return next('/login')           // 未登录
  const roles = to.meta?.roles
  if (roles && !roles.some(r => userStore.roles.includes(r)))
    return next('/403')                                  // 无权限
  next()
})
```

### 4.3 菜单按角色渲染（MainLayout.vue）
侧边栏用 `el-menu`，遍历路由表，按 `userStore.roles` 过滤出当前角色可见的菜单项：

```vue
<el-menu router>
  <el-menu-item v-for="m in visibleMenus" :key="m.path" :index="m.path">
    {{ m.meta.title }}
  </el-menu-item>
</el-menu>
```

```js
const visibleMenus = computed(() =>
  menuRoutes.filter(m => m.meta.roles.some(r => userStore.roles.includes(r)))
)
```

### 4.4 按钮级权限
对「审核」「锁价」「删除」等敏感按钮，用 `v-if` 控制：


## 六、各功能页面实现细节

### 6.1 登录页（Login.vue）—— 功能 1.x
- `el-form` 收集账号/密码，`rules` 做必填校验。
- 提交调用 `userStore.login()`；成功 `router.push('/')`，失败由拦截器弹错误（对应 1.3 登录失败提示）。
- 登录成功后 Pinia 存 token + 角色，菜单据角色渲染（1.5）。

### 6.2 供应商管理（SupplierList.vue）—— 功能 2.x
标准「查询 + 表格 + 弹窗表单」页面，是所有档案类页面的模板：

```vue
<script setup>
import { ref, reactive, onMounted } from 'vue'
import { supplierApi } from '@/api/supplier'
import { ElMessage, ElMessageBox } from 'element-plus'

const query = reactive({ current: 1, size: 10, name: '', status: null })
const tableData = ref([]); const total = ref(0); const loading = ref(false)

async function loadData() {
  loading.value = true
  try {
    const res = await supplierApi.page(query)   // 拦截器已拆出 data
    tableData.value = res.records; total.value = res.total
  } finally { loading.value = false }
}
onMounted(loadData)

// 新建/编辑弹窗
const dialogVisible = ref(false)
const form = reactive({ id: null, name: '', contactPerson: '', phone: '', creditRating: 'B' })
function openCreate() { Object.assign(form, { id: null, name: '' }); dialogVisible.value = true }
async function submit() {
  form.id ? await supplierApi.update(form.id, form) : await supplierApi.create(form)
  ElMessage.success('保存成功'); dialogVisible.value = false; loadData()
}
async function remove(row) {
  await ElMessageBox.confirm('确认删除？')
  await supplierApi.remove(row.id); loadData()
}
async function audit(row, pass) {           // 管理员审核（2.4）
  await supplierApi.audit(row.id, pass); loadData()
}
</script>
```

- 表格列：编码、名称、联系人、电话、信用评级、状态（`el-tag`）、操作。
- 采购员看到「新建/编辑」；管理员额外看到「审核/终止·启用」按钮（`v-if` 控制）。
- 分页用 `el-pagination`，`@current-change` 改 `query.current` 后重查。

### 6.3 零件管理（PartList.vue）—— 功能 3.x
- 同供应商页范式；筛选项增加「分类」「供应商」下拉。
- 表单中「采购单价」对采购员**只读**，仅管理员可通过「锁价」按钮调 `partApi.updatePrice`（3.4）。
- 「设库存预警值」可在零件表单里编辑 `stockWarningValue`（3.5）。

### 6.4 采购单（PurchaseOrderList.vue + PurchaseOrderEdit.vue）—— 功能 4.x
列表页：
- 按状态 Tab/下拉筛选（采购中/待入库审核/已入库/已取消），状态用 `ORDER_STATUS` 渲染。
- 操作按钮按状态与角色动态可用：
  - 采购中 → 「录物流」「提交入库」「取消」（采购员）。
  - 待入库审核 → 管理员在审核中心处理。
  - 已入库 → 采购员「标记已付款」。

编辑页（创建/明细）：
- 主表单：选供应商、填预计到货等。
- **明细子表**：`el-table` 内嵌可编辑行，选零件（带出锁定单价）、填数量；
- **总价自动计算**：`computed` 实时 `Σ(数量×单价)` 展示，提交后由后端落库（4.2）。

```js
const details = ref([{ partId: null, quantity: 1, unitPrice: 0 }])
const totalAmount = computed(() =>
  details.value.reduce((s, d) => s + d.quantity * d.unitPrice, 0))
function onPartChange(row) {            // 选零件带出锁定单价
  const p = partOptions.value.find(x => x.id === row.partId)
  row.unitPrice = p ? p.purchasePrice : 0
}
```

- 「录物流」弹窗：物流公司、运单号、发货时间、预计/实际到货（4.4/4.5）。
- 「提交入库」：调 `purchaseOrderApi.submitInbound(id)`，状态 1→2（4.6）。
- 「标记已付款」：调 `markPaid(id)`，`paid`→1，与状态独立（4.7）。

### 6.5 领用单（StockOutList.vue）—— 功能 5.x
- 仅领用人员/管理员可见。
- 创建：选零件 + 填领用数量 + 用途（生产/维修/报废/其他）+ 部门，明细子表同采购单。
- 「提交出库申请」：状态保持/进入待审核（5.3）。
- 列表按状态（待审核/已出库/已取消）筛选展示。

### 6.6 审核中心（AuditCenter.vue）—— 功能 6.x（管理员专属）
- 两个 Tab：**待入库审核**（采购单 status=2）、**待出库审核**（领用单 status=1）——对应 6.4 待办列表。
- 每行「通过 / 驳回」：
  - 通过入库 → `auditApi.approveInbound(orderId)`，成功后库存 +（后端事务，6.1）。
  - 通过出库 → `auditApi.approveOutbound(stockOutId)`，后端先验库存再扣减（6.2）；若库存不足，后端返回错误，前端 `ElMessage` 提示。
  - 驳回 → 弹框填原因 `auditApi.reject(...)`（6.3）。
- 审核成功后刷新待办列表与库存数据。

### 6.7 库存管理（InventoryList.vue）—— 功能 7.x
- 表格展示：零件、当前数量、安全库存、预警值、仓库位置、最近出入库时间。
- **库存预警高亮（7.3）**：用 `row-class-name` 或单元格 `el-tag`，当 `current_quantity <= warning` 标红「严重不足」、`<= safety` 标橙「不足」、否则「充足」。
- **只读保护（7.5）**：页面**不提供**任何「直接改库存数量」的输入框/按钮；数量只读展示。
- 库存预警参数配置（7.4）：仅管理员可编辑 `safetyStock/stockWarningValue`。

```vue
<el-table :data="list" :row-class-name="warnRow">
  <el-table-column prop="partName" label="零件" />
  <el-table-column prop="currentQuantity" label="当前数量" />
  <el-table-column label="状态">
    <template #default="{ row }">
      <el-tag :type="stockTag(row)">{{ stockText(row) }}</el-tag>
    </template>
  </el-table-column>
</el-table>
```

### 6.8 用户管理（UserList.vue）—— 功能 8.x（管理员专属）
- 列表 + 新建/编辑弹窗（账号、姓名、部门、电话、邮箱）。
- 「分配角色」下拉：采购员/领用人员/管理员（8.2）。
- 「启用/禁用」开关 `el-switch`（8.3）；「重置密码」按钮（8.4）。
- 按角色/状态筛选（8.5）。

### 6.9 统计报表（Dashboard.vue）—— 功能 9.x
- 采购统计（9.1）：折线/柱状图（可接 ECharts）展示按日订单数、总金额、均价。
- 库存预警汇总（9.2）：饼图「严重不足/不足/充足」数量。
- 采购明细全信息（9.3）：表格查询（对接 `order_detail_full_view`）。
- 领用出库统计（9.4）：按部门/用途/时间聚合表格。

## 七、公共交互约定

| 约定 | 实现 |
|---|---|
| 成功/失败提示 | `ElMessage`，错误由 axios 拦截器统一弹出 |
| 危险操作二次确认 | `ElMessageBox.confirm`（删除、审核驳回） |
| 表单校验 | `el-form` 的 `rules`，提交前 `formRef.validate()` |
| 加载态 | 表格 `v-loading`、按钮 `:loading` |
| 分页 | `el-pagination`，`current/size` 同步到查询对象 |
| 时间格式 | 后端返回 `yyyy-MM-dd'T'HH:mm:ss.SSS`，前端用 dayjs 格式化展示 |
| 401 处理 | 拦截器跳登录页并清 token |

## 八、与后端联调要点

1. 所有接口走 `/api/**`，开发期由 Vite 代理或直连 8080。
2. 严格按后端真实路径调用（如零件是 `/parts/page`、`/parts/update/{id}`、`/parts/update-price/{id}`）。
3. 响应统一 `Result`，拦截器只把 `data` 抛给页面，`code!=200` 一律走错误分支。
4. 状态值（数字）由前端 `enums.js` 翻译成中文标签，**不要在页面里散落魔法数字**。
5. 库存数量前端只读；任何"改库存"都通过「审核中心」触发后端事务，禁止前端直接提交库存数值。
