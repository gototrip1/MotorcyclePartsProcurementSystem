## 一、总体实现架构

### 1.1 分层职责

```
请求 → Controller(REST 入口/参数校验/组装Result)
        → Service 接口 → ServiceImpl(业务逻辑/事务/库存闸门)
            → Mapper(继承 BaseMapper, MyBatis Plus 单表 CRUD)
                → MySQL(motorparts)
```

| 层 | 包路径 | 职责 | 关键注解/父类 |
|---|---|---|---|
| 控制层 | `controller` | 暴露 `/api/**` 接口、参数校验、调用 Service、用 `Result` 包装返回 | `@RestController`、`@RequestMapping`、`@RequiredArgsConstructor`、`@Validated` |
| 业务接口 | `service` | 定义业务方法签名 | 继承 `IService<T>` |
| 业务实现 | `service/impl` | 业务逻辑、条件构造、**事务、库存变动闸门** | `@Service`，继承 `ServiceImpl<Mapper,Entity>` |
| 持久层 | `mapper` | 单表 CRUD（由 MyBatis Plus 提供），复杂查询写 XML/注解 | `@Mapper`，继承 `BaseMapper<T>` |
| 实体层 | `entity` | 与表一一映射 | 继承 `BaseEntity` |
| DTO | `dto` | 多表组合查询/请求参数对象 | —— |
| 公共 | `common` | `Result`、`PageResult`、`enums`、`exception` | —— |
| 配置 | `config` | MyBatisPlus 分页、Swagger、CORS、DDL 初始化 | `@Configuration` |

### 1.2 实现范式总览（贯穿所有模块）
- **构造器注入**：`@RequiredArgsConstructor` + `private final XxxService`，不用 `@Autowired` 字段注入。
- **统一返回**：所有接口返回 `Result<T>`，成功 `Result.success(data)`，失败 `Result.error(ResultCode.XXX)`。
- **分页**：用 MyBatis Plus 的 `Page<T>`，再 `PageResult.from(page)` 转出。
- **逻辑删除**：实体继承 `BaseEntity`，`deleted` 字段带 `@TableLogic`，`removeById` 即逻辑删。
- **条件查询**：`LambdaQueryWrapper<T>` + 方法引用（如 `Part::getName`），避免硬编码列名。

## 二、数据库连接实现

### 2.1 数据源与连接池（application.yml）

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/motorparts?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
    username: root
    password: 123123
    hikari:                       # Spring Boot 默认连接池 HikariCP
      maximum-pool-size: 20       # 最大连接数
      minimum-idle: 5             # 最小空闲连接
      connection-timeout: 30000
      idle-timeout: 600000
      max-lifetime: 1800000
      connection-test-query: SELECT 1
```

> ⚠️ **库名说明**：本系统数据库定稿名为 `motorparts`（见 [database_schema.sql](database_schema.sql)）。当前仓库的 application.yml 连接串实为 `motorparts_db`，上线前请把 url 统一改为 `.../motorparts`，与建库脚本一致。

### 2.2 MyBatis Plus 配置

```yaml
mybatis-plus:
  global-config:
    db-config:
      ddl-auto: none              # 不由框架建表，以 database_schema.sql 为准
  configuration:
    map-underscore-to-camel-case: true   # user_name -> userName 自动映射
    use-generated-keys: true              # 取回自增主键
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl   # 控制台打印 SQL
```

分页插件在 `config/MyBatisPlusConfig` 中注册 `PaginationInnerInterceptor`，使 `Page<T>` 生效。

### 2.3 表结构来源
`ddl-auto: none`，表结构不由实体反向生成，而是执行 [database_schema.sql](database_schema.sql) 建库建表。启动时 `init/DatabaseInitializer` 依据 `motorparts.init.*` 开关决定是否灌入样例数据。

### 2.4 实体与表映射（BaseEntity）
所有实体继承 `BaseEntity`，自动带三个公共字段：

```java
public abstract class BaseEntity implements Serializable {
    @TableLogic
    @TableField(value = "deleted", fill = FieldFill.INSERT)
    private Integer deleted;                 // 逻辑删除：0未删/1已删

    @TableField(value = "create_time", fill = FieldFill.INSERT)
    private LocalDateTime createTime;

    @TableField(value = "update_time", fill = FieldFill.INSERT_UPDATE)
    private LocalDateTime updateTime;
}
```

子实体示例（与 `part` 表映射）：

```java
@TableName("part")
public class Part extends BaseEntity {
    @TableId(type = IdType.AUTO)
    private Long id;
    private String partCode;        // part_code
    private String name;
    private BigDecimal purchasePrice;   // purchase_price（管理员锁定）
    private Long supplierId;        // supplier_id
    private String category;
    // ... getter/setter
}
```

## 三、公共基础设施实现

### 3.1 统一响应 Result<T>
位于 [Result.java](../src/main/java/com/motorparts/common/Result.java)，字段 `code / message / data / timestamp`，提供静态工厂：

```java
Result.success(data);              // 200 + 数据
Result.error();                    // 通用失败
Result.error(ResultCode.DATA_EXISTS);   // 指定错误码
Result.paramError("数量必须>0");    // 参数错误
```

### 3.2 分页结果 PageResult<T>
把 MyBatis Plus 的 `Page<T>` 转成对前端友好的结构：

```java
return Result.success(PageResult.from(pageResult));
// PageResult 含 records / total / current / size 等字段
```

### 3.3 枚举集中（common/enums）
- `UserRoleEnum`：角色（目标定稿为 `admin/purchase/requisition`）。
- `OrderStatusEnum`：采购单状态（1-采购中 / 2-待入库审核 / 3-已入库 / 4-已取消）。
- `SupplierStatusEnum`：供应商状态（1-合作中 / 2-已终止 / 3-审核中）。
- `PartCategoryEnum`：零件分类。
- `ResultCode`：响应码（SUCCESS / ERROR / PARAM_ERROR / DATA_EXISTS / DATA_NOT_EXISTS / UNAUTHORIZED / FORBIDDEN ...）。

### 3.4 全局异常处理
`common/exception/GlobalExceptionHandler` 用 `@RestControllerAdvice` 捕获：
- `BusinessException` → 返回业务错误码与消息；
- `MethodArgumentNotValidException`（校验失败）→ `Result.paramError(...)`；
- 其它 `Exception` → `Result.serverError()`。

### 3.5 对外发布 RESTful 接口的机制
1. `@RestController` + `@RequestMapping("/api/xxx")` 声明资源根路径。
2. 方法上用 `@PostMapping/@GetMapping/@PutMapping/@DeleteMapping/@PatchMapping` 映射动作。
3. `@RequestBody` 收 JSON 体、`@PathVariable` 收路径参数、`@RequestParam` 收查询参数。
4. `@Valid` 触发 Bean Validation 校验。
5. 返回值统一 `Result<T>`，由 Spring 自动序列化为 JSON。
6. **跨域**：`config/WebMvcConfig` 对 `/**` 放开 CORS，前端 axios 可直连 8080。
7. **文档**：`@Tag`/`@Operation`/`@Parameter` 生成 Swagger UI（`/swagger-ui.html`）。

### 4.1 Mapper 层（最薄）


### 4.2 Service 接口

```java

    List<Part> searchByName(String name);
```

### 4.3 ServiceImpl 实现（条件构造在这里）

```java
@Service
public class PartServiceImpl extends ServiceImpl<PartMapper, Part> implements PartService {

    @Override
    public Page<Part> pageParts(Page<Part> page, String name, String category, Long supplierId) {
        LambdaQueryWrapper<Part> wrapper = new LambdaQueryWrapper<>();
        if (StringUtils.hasText(name))     wrapper.like(Part::getName, name);
        if (StringUtils.hasText(category)) wrapper.eq(Part::getCategory, category);
        if (supplierId != null)            wrapper.eq(Part::getSupplierId, supplierId);
        wrapper.orderByDesc(Part::getCreateTime);
        return baseMapper.selectPage(page, wrapper);     // 自动带 deleted=0 过滤
    }

    @Override
    public boolean updatePurchasePrice(Long id, BigDecimal purchasePrice) {
        if (id == null || purchasePrice == null || purchasePrice.compareTo(BigDecimal.ZERO) < 0) return false;
        LambdaUpdateWrapper<Part> wrapper = new LambdaUpdateWrapper<>();
        wrapper.set(Part::getPurchasePrice, purchasePrice).eq(Part::getId, id);
        return update(wrapper);
    }

    @Override
    public boolean existsPartCode(String partCode, Long excludeId) {
        LambdaQueryWrapper<Part> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(Part::getPartCode, partCode);
        if (excludeId != null) wrapper.ne(Part::getId, excludeId);
        return baseMapper.selectCount(wrapper) > 0;
    }
}
```

### 4.4 Controller 层（对外接口）

```java
@Tag(name = "产品/零部件管理")
@RestController
@RequestMapping("/api/parts")
@RequiredArgsConstructor
@Validated
public class PartController {

    private final PartService partService;     // 构造器注入

    @Operation(summary = "创建零部件")
    @PostMapping("/create")
    public Result<Part> create(@Valid @RequestBody Part part) {
        if (partService.existsPartCode(part.getPartCode(), null)) {
            return Result.error(ResultCode.DATA_EXISTS);     // 编码重复
        }
        part.setDeleted(0);
        part.setCreateTime(LocalDateTime.now());
        part.setUpdateTime(LocalDateTime.now());
        return partService.save(part) ? Result.success(part) : Result.error();
    }

    @Operation(summary = "分页查询零部件")
    @GetMapping("/page")
    public Result<PageResult<Part>> page(
            @RequestParam(defaultValue = "1") Long current,
            @RequestParam(defaultValue = "10") Long size,
            @RequestParam(required = false) String name,
            @RequestParam(required = false) String category,
            @RequestParam(required = false) Long supplierId) {
        Page<Part> page = new Page<>(current, size);
        Page<Part> result = partService.pageParts(page, name, category, supplierId);
        return Result.success(PageResult.from(result));
    }

    @Operation(summary = "更新采购单价")     // 管理员锁价
    @PatchMapping("/update-price/{id}")
    public Result<Void> updatePrice(@PathVariable Long id, @RequestParam BigDecimal purchasePrice) {
        return partService.updatePurchasePrice(id, purchasePrice)
                ? Result.success() : Result.error(ResultCode.DATA_NOT_EXISTS);
    }
    // update / delete / get / search / supplier 同理……
}
```

## 五、各业务模块实现要点

> 标注【已实现】= 仓库现有代码；【按设计补充】= 目标定稿需新增的实现（沿用第四章同一范式）。

### 5.1 供应商管理【已实现】（`SupplierController`）
- 同零件范式：`create / update / delete / get / page`。
- 业务差异点：新建时 `status` 置 **3-审核中**；管理员审核接口把 `status` 改为 1-合作中 或驳回；可单独提供「终止/启用合作」的状态切换接口（`@PatchMapping`）。

### 5.2 零件管理【已实现】
见第四章，重点是 `update-price` 仅供管理员锁价。

### 5.3 采购入库管理【已实现】（`PurchaseOrderController` + `PurchaseOrderServiceImpl`）
核心难点是「**主单 + 明细 + 总价**」一致性，需用事务：

```java
@Service
public class PurchaseOrderServiceImpl extends ServiceImpl<PurchaseOrderMapper, PurchaseOrder>
        implements PurchaseOrderService {

    private final OrderDetailMapper orderDetailMapper;

    @Transactional(rollbackFor = Exception.class)
    public Long createOrder(PurchaseOrderWithDetails dto) {
        PurchaseOrder order = dto.getOrder();
        order.setStatus(1);          // 采购中
        order.setPaid(0);            // 未付款
        save(order);                 // 取回自增 id
        BigDecimal total = BigDecimal.ZERO;
        for (OrderDetail d : dto.getDetails()) {
            d.setOrderId(order.getId());
            // unit_price 取零件锁定单价；subtotal 由数据库生成列自动算
            orderDetailMapper.insert(d);
            total = total.add(d.getUnitPrice().multiply(BigDecimal.valueOf(d.getQuantity())));
        }
        order.setTotalAmount(total); // 自动计算订单总价（功能 4.2）
        updateById(order);
        return order.getId();
    }
}
```

- **录物流（4.4/4.5）**：`@PatchMapping` 更新 `logistics_company/tracking_number/ship_time/expected/actual_delivery_date`（字段已并入 purchase_order）。
- **提交入库（4.6）**：把 `status` 由 1→2（待入库审核），**不碰库存**。
- **标记付款（4.7）**：`@PatchMapping` 把 `paid` 置 1，与 status 互不干扰。
- **取消（4.8）**：`status` → 4。
- **列表（4.9）**：`LambdaQueryWrapper` 按 `status`、时间、创建人筛选 + 分页。

### 5.4 库存管理【已实现】（`InventoryController`）
- 查询：`page` 联零件信息返回 `InventoryWithPart`（DTO）。
- **库存预警（7.3）**：查 `inventory_warning_view` 视图，或在 Service 里比较 `current_quantity` 与 `stock_warning_value/safety_stock` 标记「严重不足/不足/充足」。
- **只读保护（7.5）**：**不提供**任何直接写 `current_quantity` 的对外接口；该字段只在审核 Service 内部变更（见 5.6）。

### 5.5 领用出库管理【按设计补充】（建议新增 `StockOutOrderController`）
照采购单范式做「主单 `stock_out_order` + 明细 `stock_out_detail`」：
- `create`：建领用单，`status`=1（待审核），写明细（part_id + quantity + 用途/部门）。
- `submit`：提交出库申请（保持/进入待审核态）。
- `cancel`：`status`→3。
- `page`：按状态/部门/领用人筛选。
- **同样不碰库存**，库存减少留给审核中心。

### 5.6 审核中心【按设计补充，库存变动唯一入口】（建议新增 `AuditController` + 审核 Service）
这是「**库存闸门**」，唯一允许写 `inventory.current_quantity` 的地方，必须事务化：

```java
@Service
@RequiredArgsConstructor
public class AuditServiceImpl implements AuditService {

    private final PurchaseOrderService purchaseOrderService;
    private final OrderDetailMapper orderDetailMapper;
    private final StockOutOrderService stockOutOrderService;
    private final StockOutDetailMapper stockOutDetailMapper;
    private final InventoryMapper inventoryMapper;

    /** 审核采购入库：通过 → 库存 +  (功能 6.1) */
    @Transactional(rollbackFor = Exception.class)
    public void approveInbound(Long orderId) {
        PurchaseOrder order = purchaseOrderService.getById(orderId);
        if (order == null || order.getStatus() != 2)        // 必须是“待入库审核”
            throw new BusinessException("订单状态不允许入库");
        for (OrderDetail d : orderDetailMapper.selectByOrderId(orderId)) {
            // 库存 += 数量，并更新最近入库时间（原子 update）
            inventoryMapper.increaseQuantity(d.getPartId(), d.getQuantity(), LocalDateTime.now());
        }
        order.setStatus(3);                                  // 已入库
        purchaseOrderService.updateById(order);
    }

    /** 审核领用出库：先验库存 → 通过 → 库存 −  (功能 6.2) */
    @Transactional(rollbackFor = Exception.class)
    public void approveOutbound(Long stockOutId) {
        StockOutOrder so = stockOutOrderService.getById(stockOutId);
        if (so == null || so.getStatus() != 1)
            throw new BusinessException("领用单状态不允许出库");
        for (StockOutDetail d : stockOutDetailMapper.selectByOrderId(stockOutId)) {
            Inventory inv = inventoryMapper.selectByPartId(d.getPartId());
            if (inv.getCurrentQuantity() < d.getQuantity())  // 先验证库存充足
                throw new BusinessException("库存不足，零件ID=" + d.getPartId());
        }
        for (StockOutDetail d : stockOutDetailMapper.selectByOrderId(stockOutId)) {
            inventoryMapper.decreaseQuantity(d.getPartId(), d.getQuantity(), LocalDateTime.now());
        }
        so.setStatus(2);                                     // 已出库
        stockOutOrderService.updateById(so);
    }

    /** 驳回：仅改状态、附原因，库存不变 (功能 6.3) */
    public void reject(...) { /* 状态回退 + remark */ }
}
```

库存原子更新的 Mapper（防并发超扣，用带条件的 update）：

```java
@Mapper
public interface InventoryMapper extends BaseMapper<Inventory> {

    @Update("UPDATE inventory SET current_quantity = current_quantity + #{qty}, " +
            "last_inbound_time = #{time} WHERE part_id = #{partId}")
    int increaseQuantity(@Param("partId") Long partId, @Param("qty") int qty, @Param("time") LocalDateTime time);

    @Update("UPDATE inventory SET current_quantity = current_quantity - #{qty}, " +
            "last_outbound_time = #{time} WHERE part_id = #{partId} AND current_quantity >= #{qty}")
    int decreaseQuantity(@Param("partId") Long partId, @Param("qty") int qty, @Param("time") LocalDateTime time);
}
```

> `decreaseQuantity` 的 `AND current_quantity >= #{qty}` 在数据库层兜底防超扣；返回 0 行即视为库存不足，回滚事务。

## 六、复杂查询与 DTO

当单表查询无法满足（需 join 零件、供应商等），在 Mapper 声明方法并用 `@Select` 或 XML，返回 DTO：

| DTO | 用途 |
|---|---|
| `PartWithSupplier` | 零件 + 所属供应商名 |
| `InventoryWithPart` | 库存 + 零件信息（库存列表展示） |
| `OrderDetailWithPartAndSupplier` | 采购明细 + 零件 + 供应商（对应 `order_detail_full_view`） |
| `PurchaseOrderWithDetails` | 采购主单 + 明细列表（创建/详情用） |

## 七、事务与一致性

| 场景 | 事务边界 | 要点 |
|---|---|---|
| 创建采购单 + 明细 + 总价 | `createOrder` 整方法 `@Transactional` | 主单与多条明细要么全成功 |
| 审核入库（库存 +） | `approveInbound` | 改库存 + 改单据状态同一事务 |
| 审核出库（库存 −） | `approveOutbound` | 先验库存 → 扣减 → 改状态；DB 层 `>= qty` 兜底 |
| 逻辑删除 | 单条 | `@TableLogic` 自动转 update deleted=1 |

`@Transactional(rollbackFor = Exception.class)` 保证任何异常都回滚，库存与单据状态不会出现"半变更"。

## 八、对外接口总览（按模块）

| 模块 | 典型接口前缀 | 实现状态 |
